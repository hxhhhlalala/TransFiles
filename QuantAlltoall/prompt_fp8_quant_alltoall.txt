【角色定义】
你是一位深耕 AI 芯片推理优化的高级系统工程师，精通 Ascend NPU 编程模型、
PyTorch/torch_npu 算子、Ulysses 序列并行通信框架，以及 FP8 低精度推理技术。
你擅长将 profiler 原始数据转化为可落地的优化方案，并以结构化技术文档的形式
输出，供团队直接参考实施。

【背景与上下文】
项目：Wan2.2 视频生成模型，运行于 Ascend NPU，序列并行度 SP=4。
核心组件：
  - 注意力模块：xFuserLongContextAttention（基线）
               QuantAllToAllAttention（FP8 子类，QUANT_ALLTOALL=1 激活）
  - 通信原语：all_to_all_4D(scatter_idx, gather_idx, group)
  - FP8 量化：fa_block_quant_preprocess(layout, block_size, dst_type)
  - FP8 FA kernel：npu_fused_infer_attention_score_v2(quant_mode=7, dequant_scale_*)

已知 profiler 实测数据（同一 Transformer 层，SP=4，N=40，D=128）：
  Before（BF16 BSND A2A）：
    - QKV A2A 三路合计：~3,010 µs
    - per-head 量化（×10）：~1,310 µs（在 FA 循环内）
    - 量化预处理（循环外）：0 µs
  After（FP8 BSND A2A）：
    - QKV A2A 三路合计：~1,507 µs（降幅 50%，与理论 2× 一致）
    - per-head 量化：0 µs（已移至循环外）
    - 量化预处理（一次性）：~1,158 µs
    - post-A2A 转置（BSND→BNSD）：~243 µs
    - Scale A2A 额外开销：~15 µs

代码结构约束：
  - FP8 路径封装于 QuantAllToAllAttention 子类，不修改基线类
  - 调用点：sequence_parallel.py 中 sp_attn_forward，按 QUANT_ALLTOALL 环境变量选路
  - 非 overlap 路径：循环外整体旋转+量化 → FP8 A2A → scale A2A → per-head FA
  - Overlap 路径：FP8 量化 → chunk 切分 → 双流流水（当前实测无法有效 overlap，
    原因：A2A 批量同步执行，FA[1-9] 无流水线重叠）

【任务说明】
请完成以下工作（请按编号逐步推进）：

1. 分析：基于 profiler 数据，量化 Before/After 在各阶段的耗时差异，计算净收益，
   解释为何"量化开销从循环内移至循环外"能带来整体加速。

2. 原理说明：解释为何最终选择 BSND（scatter_idx=2, gather_idx=1）而非 BNSD
   方向，从 all_to_all_4D 内部 transpose 次数角度给出量化分析。

3. 数据流对比：用 Markdown 表格并列展示 Before 与 After 完整数据流（输入 →
   循环外预处理 → A2A → 转置 → per-head FA → cat → 输出 A2A），
   每一步注明张量形状、数据类型和关键函数名（使用原始接口名，不简写）。

4. 代码结构说明：用类继承树和伪代码展示 QuantAllToAllAttention 的架构，
   包括 forward 的路由逻辑（何时回退到 super().forward()）。

5. Overlap 问题根因分析：对比 Before/After 双流流水线时间线，
   说明 After 为何退化为串行，并给出改进方向。

6. 接入 Checklist：按"代码结构"和"算法正确性"两组列出检查项。

  → 以上推导过程不输出，直接输出结论性内容。

- 所有函数名/接口名使用原始完整名称（如 npu_fused_infer_attention_score_v2，
  不得缩写为 FA_v2 或 _fp8_attn）
- 张量维度描述统一使用 [B, S, N, D] 或 [B, N, S, D] 格式，并注明布局名（BSND/BNSD）
- profiler 耗时数据保留至 µs 精度，不做四舍五入
- 关键结论必须有数据支撑，不得出现"可能""大概"等模糊措辞
- 代码示例使用 Python + torch_npu 真实 API，可直接复制运行
- 文档结构：使用 Markdown，包含章节编号、表格、代码块、注意事项引用块（>）
