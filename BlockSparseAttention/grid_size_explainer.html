03 / 08

Grid Size = 一个三维棋盘格

把视频想象成一个由小格子构成的立方体：T 层（时间）× H 行（高）× W 列（宽）

（示意图，按比例缩放显示）

🕐 时间维度 T = 21

代表 21 个时刻的视频帧
原始 81 帧经 VAE 时序压缩 ÷4 得到

↕ 高度维度 H = 69

视频帧被切成 69 个横向 patch
每个 patch 对应原图 16×16 像素区域

↔ 宽度维度 W = 52

视频帧被切成 52 个纵向 patch
每帧共 69×52 = 3,588 个 patch

第二步：把 Token 序列切成 Block

75,348 个 token 排成一列，每 128 个 token 打包成一个 Block，用于稀疏计算。

75,348 tokens（每个小方块 = 128 tokens）
共 589 个 Block（ceil(75348 / 128)）
其中各部分分布：
首帧 (69×52=3588 → 28 blocks)
其余 20 帧 (71760 → 561 blocks)

🔢 数学关系

Token 总数 = T × H × W
= 21 × 69 × 52 = 75,348
Block 数 = ⌈75348 / 128⌉ = 589
掩码矩阵大小
= 589 × 589 = 346,921 格
Shape: [B=1, N=20, 589, 589]
(每张卡 N=40÷2=20 heads，Ulysses SP=2)

首帧保护：参考帧的特殊地位

I2V 模型中，第 0 帧是输入的参考图像，是后续帧运动的"锚点"。 即使稀疏度很高，所有帧也必须能看到第 0 帧。

🖼️
第 0 帧
参考图像 (i2v input)
🔒 永远保留
28 blocks 全为 1
🎞️
第 1~4 帧
latent t=1~4
⚡ 稀疏计算
🎞️
第 5~12 帧
latent t=5~12
⚡ 稀疏计算
🎞️
第 13~20 帧
latent t=13~20
⚡ 稀疏计算

在代码中的体现

firstframe_block_num = ceil(69×52 / 128) = 28
mask[:, :, :28, :] = 1  # 首帧对应的 query 行强制保留
mask[:, :, :, :28] = 1  # 首帧对应的 key 列强制保留

Mask 缓存复用：省掉重复计算

去噪过程有 20 步，视频内容结构在相邻步之间变化很小。 可以复用上一步的 mask，省去 pool + einsum + topk 约 863µs/步。

⚡ 节省的操作（每次 cache hit）

avgpool (AsStrided+ReduceMean)
~ 746 µs
Mask 生成 (MatMul→softmax→topk)
~ 117 µs
S-padding × 3 (已移除)
~ 454 µs
合计节省（含去 padding）
≥ 863 µs/步

总结：Grid Size 的完整含义

grid_size = [21, 69, 52] 告诉模型这段视频在 latent 空间的三维结构，是空间稀疏注意力的基础。

🗺️

空间重排依据

rearrange_with_remaining 按 (h//8, w//8) 把 patch 重组为 8×8 的局部块，提升相邻 token 的物理连续性

🔒

首帧保护范围

grid_size[1] × grid_size[2] = 3588 tokens → 28 blocks 永远全为 1，保证参考帧可见性

📦

Mask 矩阵尺寸

T×H×W = 75348 tokens → 589 blocks → 589×589 int8 mask，约 335KB（每卡每层）

♻️

逆重排恢复

_bsa_inv_rearrange 用 grid_size 做 (tq, hn, wn, 8, 8) → (tq, hn, 8, wn, 8) 的 permute，还原 token 原始顺序

🏁
一句话总结
Grid Size 就是你的视频在"AI 眼中"的三维地图坐标系， 模型用它来切块、排序、跳过不重要的注意力，实现高效推理。
