Step 1：AvgPool —— 把 Token 序列"粗粒化"

原始序列有 75,348 个 token，直接计算两两 attention 太贵。先用平均池化把每 128 个 token 压缩成 1 个"代表向量"，只用 589 个代表 来决定哪些 block 重要。

🔍 池化的直觉

就像把一段视频先缩略成缩略图，用缩略图来判断两帧之间是否"相关"，再决定是否值得精细计算。

// BSND input: [B, S, N, D]
pool_shape = [B, S÷128, N, D]

// 对每个 block 内取平均
q_pool[b, i, n, :] =
mean( q[b, i*128:(i+1)*128, n, :] )

// 75348 tokens → 589 代表向量
75348 → 589  (÷ 128×)
💡 为什么用平均而不是最大值？

平均池化保留了整个 block 的"平均语义"，对噪声更鲁棒；最大值会被极端 token 主导，导致 mask 不稳定。

03 / 08

Step 2：计算 Block Score 矩阵

用池化后的 Q 和 K 做点积，得到 589×589 的得分矩阵——每个格子代表"Query Block i 有多需要关注 KV Block j"。

① Einsum 点积

score = einsum(
  "blnd, bsnd → bnls",
  q_pool, k_pool
) × scale
// shape: [B, N, 589, 589]

每个 attention head 独立计算，得到各自的 block 重要性视图。

② Row-wise Softmax

对每一行（每个 Query Block）做 softmax，将得分归一化为概率分布，表示"该 Query 的注意力预算分配"。

score = softmax(score, dim=-1)
// 每行和为 1

③ 结果含义

矩阵中亮色格 = 高相关度（值接近 1），暗格 = 低相关度（接近 0）。下一步用 TopK 选出最重要的亮格保留。

04 / 08

Step 3：TopK 阈值 —— 决定丢弃哪些 Block

对每行按得分排序，保留得分最高的 ceil(kv_blocks × (1 - sparsity)) 个 block，其余置 0。
拖动下方滑块，直观感受不同稀疏度的效果。

0.64 （0 = 全保留，0.95 = 仅保留 5%）
保留的 Block 比例（每行）
保留 / 589 blocks
裁剪的 Block 比例
丢弃 / 589 blocks

🔢 TopK 选择逻辑

keep = ceil(589 × (1 - sparsity))

topk_vals, _ = topk(score, k=keep)
threshold = topk_vals[..., -1:]
mask = score >= threshold
// bool → int8 [B,N,589,589]

阈值是第 keep 大的得分值——得分 ≥ 阈值的 block 保留，低于阈值的置 0。每行独立计算，不同 Query Block 保留的 KV 集合可以不同。

05 / 08

首帧保护：不可被稀疏掉的锚点

生成掩码后，强制将首帧对应的行与列全部覆写为 1——无论 sparsity 多高，参考帧永远可见。

为什么首帧必须保留？

I2V 模型中第 0 帧是用户输入的参考图像，是所有后续帧的"运动起点"。若某帧的 Query 无法关注到参考帧的 Key，模型将失去对原始内容的约束，导致生成结果漂移或与输入不一致。

代码实现

first_blocks = ceil(H×W / 128)
# = ceil(69×52 / 128) = 28

mask[:, :, :28, :] = 1
# ↑ 所有 Query 都能看首帧 KV

mask[:, :, :, :28] = 1
# ↑ 首帧自身也能关注所有 KV

覆写发生在 TopK 之后

TopK 先按内容重要性生成稀疏 mask，然后无条件覆写首帧区域。这样既保证了内容驱动的稀疏化，又保证了参考帧的绝对可见性，两者不互斥。

06 / 08

完整流水线：从 Q/K/V 到输出

六个顺序步骤，每步都有对应的 tensor shape 变化。

1

空间重排（rearrange_with_remaining）

把每帧的 H×W token 从行扫描顺序重组为 8×8 的局部块顺序，使相邻 token 在序列中物理相邻，改善 pool 质量。

Q/K/V [B, T·H·W, N, D] 行扫描 → Q'/K'/V' [B, T·H·W, N, D] 块顺序
2

AvgPool（pool_size=128）

对重排后的序列做滑动均值池化，每 128 个 token 压缩为 1 个代表向量，同时 concat(Q_pool, K_pool, V_pool) 形成统一 tensor。

Q'/K'/V' [B, 75348, N, D] → qkv_pool [3B, 589, N, D]
3

Block Score 矩阵

einsum("blnd, bsnd → bnls") 对池化 Q/K 做点积并 × scale，再做 row-wise softmax，得到每个 head 的 block 注意力分布。

q_pool/k_pool [B, 589, N, D] → score [B, N, 589, 589] float
4

TopK → int8 Mask（含首帧覆写）

每行取 TopK，得到 bool mask；然后强制首帧行列为 1；最后转 int8 交给 BSA 算子。

score [B, N, 589, 589] → mask [B, N, 589, 589] int8
5

block_sparse_attention（BNSD）

算子根据 int8 mask 决定计算哪些 (Q_block_i, KV_block_j) 的注意力。mask=0 的组合完全跳过，不参与任何计算（非近似，而是精确稀疏）。

Q'/K'/V' + mask → out [B, S, N, D] BSND
6

逆重排（_bsa_inv_rearrange）

将 BSA 输出从块顺序恢复为原始行扫描顺序，使 token 位置与输入 Q/K/V 一一对应，后续模块可正常使用。

out [B, S, N, D] 块顺序 → out [B, S, N, D] 行扫描
07 / 08

skip_timesteps + Mask 缓存：两级优化

去噪过程分两个阶段：早期高噪声步用 Dense Attention 保质量，后期低噪声步用 BSA 提效率；BSA 步内还可复用 mask 进一步减少开销。

🔵

Dense 阶段（t < skip_timesteps）

噪声大，视频内容还不确定，稀疏 mask 容易选错关键 block。全量注意力确保信息完整传递。此阶段结束时清除 mask 缓存。

🟡

首个 BSA 步（重新计算 Mask）

视频结构已基本稳定，用当前 Q/K pool 生成新 mask，缓存供后续步复用。包含完整的 pool → score → topk → 首帧保护流程。

🟢

后续 BSA 步（复用 Mask）

视频结构变化极小，直接用缓存的 mask，跳过 pool + score + topk 计算。mask_refresh_interval=0 永久复用，N 则每 N 步刷新一次。

⚙️ 参数速查

skip_timesteps=5
前 5 步用 Dense
sparsity=0.64
保留 36% 的 blocks
mask_refresh_interval=0
首个稀疏步计算，之后永远复用
mask_refresh_interval=1
每步重新计算（等价于不缓存）
mask_refresh_interval=3
每 3 步刷新一次

核心概念总结

BSA 的本质是：用低成本的粗粒度 Q/K 得分来预测精细注意力的稀疏结构，以少量代理计算换取大量实际计算的跳过。

🗜️
AvgPool

128 token → 1 代表向量。用"缩略图"决策，而不是全部内容做决策，O(S²) → O((S/B)²) 的代理计算。

📐
Score 矩阵

池化 Q×K 点积 + softmax 得到 block 级注意力分布。每个 head 独立计算，支持不同 head 有不同稀疏结构。

🎯
TopK 阈值

按行取 Top-K，阈值是第 K 大的得分。高于阈值 → mask=1（计算），低于 → mask=0（跳过）。每行保留数量相同，结构规整利于硬件优化。

🔒
首帧强制保护

覆写首帧对应的行列为 1，确保 I2V 参考图像的信息不被稀疏化截断，是精度保障的关键约束。

⚡
Sparse Kernel

int8 mask 直接传给 block_sparse_attention，算子按 block 跳过计算，不是近似而是精确稀疏——mask=0 的组合真正不参与矩阵运算。

♻️
Mask 缓存

去噪相邻步的视频结构高度相似，mask 可跨步复用。缓存命中时跳过 pool+score+topk，仅做空间重排后直接调用 BSA kernel。

🏁
一句话总结
Block Sparse Attention 用"先粗看、再精跳"的策略：池化代理判断重要性 → TopK 生成稀疏 mask → Kernel 精确跳过不重要的 Block → 时序缓存避免重复判断。本质是用 O((S/B)²) 的预筛选，换取 sparsity × O(S²) 的主计算节省。
